# domain-rule-creator

현재 프로젝트의 도메인 지식 관리 틀을 `rules/domain.md`에 생성·갱신합니다. 실제 도메인 내용은 `rules/domain/` 하위 주제별 파일로 관리합니다.

## 생성 절차

1. **프로젝트 스캔** — 도메인 맥락을 파악합니다.
   - 기존 `docs/` 구조, 스펙 문서, 용어가 등장하는 README.
   - 주요 모델·엔티티 이름(코드 상위 디렉토리 구조에서 드러남).
2. **내용 구성**
   - 같은 디렉토리의 `domain.md` 전체를 기본 내용으로 사용합니다.
   - 프로젝트 고유의 도메인 경계·결정권자·용어 표기 규칙을 "## 이 프로젝트의 세부 지침"에 추가합니다.
3. **파일 기록**
   - 대상: `rules/domain.md` (메타)
   - 필요 시 `rules/domain/` 디렉토리와 주제별 스켈레톤 파일(`glossary.md`, `entities.md` 등)의 생성을 사용자에게 제안합니다. 기본으로 자동 생성하지 않습니다.
   - 이미 존재하면 덮어쓰지 않고 diff 후 사용자에게 확인.
