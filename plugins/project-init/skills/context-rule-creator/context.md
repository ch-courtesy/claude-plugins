# 컨텍스트 관리 지침

AI 에이전트·세션에서 컨텍스트(정보의 적재 위치·수명·우선순위)를 관리하는 규칙입니다. 개별 도구를 나열하는 대신, 정보를 **수명 계층**으로 분류하고 계층 간 이동 기준을 적용합니다.

## 정보 계층

각 정보를 아래 계층 중 어디에 두는지 판단하고, 계층에 맞게 관리합니다.

### 1. 휘발성 (Ephemeral)

**특성**: 현재 턴의 도구 결과, 임시 출력, 사고 과정. 다음 턴이나 재시작에 사라짐.

**규칙**: 이 계층에만 존재하는 중요한 정보는 즉시 세션 또는 프로젝트 계층으로 옮깁니다. "기억하고 있을 것"에 의존하지 않습니다.

### 2. 세션 (Session)

**특성**: 현재 대화의 메시지·읽은 파일·실행 결과. 세션 동안만 유지되고 컨텍스트 창을 초과하면 압축·폐기됨.

**규칙**:
- **이미 적재된 것**은 다시 읽거나 다시 검색하지 않습니다.
- **대규모 출력**(긴 로그, 전체 파일)은 필요 범위만 잘라 읽습니다.
- **중요 발견**은 요약·기록해 세션이 끝나거나 컨텍스트가 밀려도 남도록 합니다.

### 3. 프로젝트 (Project)

**특성**: `CLAUDE.md`, `rules/`, 소스 파일·문서. 어떤 세션에서도 접근 가능한 프로젝트 자산.

**규칙**:
- 반복 참조될 결정·컨벤션은 **프로젝트 파일**로 기록합니다. 세션에서만 공유되면 다음 세션이 다시 처음부터 시작합니다.
- 기록 위치는 해당 카테고리의 지침 파일(`rules/<category>.md`)을 우선합니다.

### 4. 영속 (Persistent)

**특성**: 메모리 파일, 외부 노트 등 세션과 프로젝트를 가로질러 유지되는 지식.

**규칙**:
- **프로젝트에 독립적인 지식**(사용자 선호, 공통 작업 방식, 크로스 프로젝트 맥락)만 영속 계층에 둡니다.
- 프로젝트 고유 내용은 프로젝트 파일(`rules/` 등)에 둡니다. 영속 계층에 중복 보관하지 않습니다.

### 5. 외부 참조 (External)

**특성**: 문서 URL, 외부 API, 타 레포. 우리 저장소에는 없지만 참조 가능한 지식.

**규칙**:
- 자주 쓰는 외부 레퍼런스는 프로젝트 파일에 **링크와 한 줄 설명**을 남겨 둡니다(본문을 통째로 복사하지 않습니다).
- 참조 시점의 버전·시간을 기록하면 이후 변경 감지에 도움이 됩니다.

## 계층 간 이동 기준

- **휘발 → 세션**: 중요한 도구 결과는 메시지 본문에 요약·기록해 이후 턴에서도 접근 가능하게 합니다.
- **세션 → 프로젝트**: 같은 결정·규칙이 두 번 이상 등장하면 프로젝트 파일로 승격합니다 (승격 기준 세부는 `rules/meta.md` 참조).
- **프로젝트 → 영속**: 프로젝트 경계를 넘어 유용한 것만 영속 계층으로 승격합니다.

## 컨텍스트 보호 (세션 계층 아끼기)

- **대규모 탐색은 위임**: 같은 주제로 3회 이상의 검색·읽기가 예상되면 서브에이전트(Explore 등)에 맡겨 주 컨텍스트를 보호합니다. 에이전트는 요약만 돌려주도록 요청합니다.
- **독립 작업은 병렬로**: 서로 의존이 없는 조사·도구 호출은 병렬로 수행해 반복 적재를 피합니다.
- **출력 최소화**: 긴 로그·원문을 그대로 들고 있지 말고 요약·핵심만 남깁니다.
- **중간 산출물은 파일로**: 검토용 긴 문서·리포트는 임시 파일(파일시스템 지침 "임시·작업용" 카테고리)로 오프로드하고 참조만 유지합니다.

## 기록 타이밍

- 중요한 도구 결과는 **삭제·압축 전에** 요약해 둡니다.
- 주요 결정은 **즉시** 관련 파일·메모리에 반영합니다. 나중으로 미루면 사라집니다.
- 긴 작업 도중 주기적으로 "지금까지 확정된 것"을 요약해 최신 상태를 유지합니다.

## 오버로드 감지

다음 신호가 보이면 컨텍스트가 비대해졌거나 흩어졌다는 뜻입니다:

- 같은 정보를 반복 검색·재확인하고 있음
- 초기 지시사항을 누락하거나 혼동함
- 이미 답한 질문을 다시 묻고 있음
- 비슷한 도구 결과가 중복 적재되어 있음

감지되면 **요약·정리·위임**으로 컨텍스트를 줄입니다.

## 공통 규칙

- **필요한 것만 적재**: 호기심이나 "혹시 모르니"로 파일·결과를 적재하지 않습니다.
- **출처를 함께 기록**: 어디서 왔는지 알아야 나중에 검증할 수 있습니다.
- **컨텍스트는 공짜가 아니다**: 토큰 예산·주의 분산은 실제 비용입니다. 적재 전에 "이게 지금 작업에 진짜 필요한가"를 묻습니다.
