# 파일시스템 지침

이 프로젝트의 파일을 다룰 때 따라야 할 규칙입니다. 개별 경로를 나열하는 대신, 파일·디렉토리를 **특성별 카테고리**로 분류하고 카테고리별 규칙을 적용합니다.

## 분류 원칙

파일·디렉토리를 건드리기 전에 아래 카테고리 중 어디에 해당하는지 판단하고, 해당 카테고리의 규칙을 따릅니다. 애매하면 사용자에게 확인합니다.

### 1. 도구 관리 영역 (Tool-managed)

**특성**: 의존성 매니저, 빌드 시스템, VCS 등 다른 도구가 생성·갱신하는 영역. 사람이 직접 편집하면 무결성이 깨지거나 재생성 시 덮어써집니다.

**규칙**:
- 손으로 수정·생성하지 않습니다.
- 변경이 필요하면 해당 도구의 인터페이스(패키지 매니저 명령, 빌드 스크립트, VCS 명령)를 통해서만 수행합니다.
- 버전 관리에서 제외되어야 하는 산출물은 `.gitignore`에 포함되었는지 확인합니다.

**예**: 의존성 설치 디렉토리, 빌드/캐시 산출물, 잠금 파일, VCS 내부 상태.

### 2. 비밀·민감 정보 (Sensitive)

**특성**: 자격증명, API 키, 토큰, 개인 환경 설정 등 외부에 노출되면 안 되는 값을 담고 있음.

**규칙**:
- 내용을 읽어들이거나 출력·로그·커밋하지 않습니다.
- 값이 필요하면 환경변수 이름만 언급하고 실제 값은 참조하지 않습니다.
- 예시 파일(`.env.example` 등)에는 실제 값 대신 placeholder만 둡니다.

**예**: `.env` 계열, 개인 키 파일, 자격증명 파일.

### 3. 작업 산출물 (Work product)

**특성**: 프로젝트의 실제 결과물 — 소스 코드, 문서, 설정 템플릿.

**규칙**:
- 프로젝트 컨벤션에 따라 자유롭게 편집합니다.
- 버전 관리 대상입니다.
- 자동 생성되는 "작업 산출물"(코드 생성기 출력 등)은 도구 관리 영역으로 취급합니다.

### 4. 임시·작업용 (Ephemeral)

**특성**: 디버그 스크래치, 일회성 출력, 로컬 테스트 결과 등 수명이 짧고 공유 가치가 없는 파일.

**규칙**:
- OS 임시 디렉토리(`$TMPDIR`)나 `.gitignore`된 `tmp/` 하위에만 생성합니다.
- 프로젝트 루트에 `scratch.*`, `test.*`, `foo.*` 같은 파일을 남기지 않습니다.
- 작업이 끝나면 정리합니다.

### 5. 로컬 전용 설정 (Local-only)

**특성**: 사용자·머신 고유의 설정이나 데이터. 공유 대상이 아님.

**규칙**:
- 버전 관리에서 제외합니다(`.gitignore`).
- 다른 사용자의 환경 구조를 코드가 전제하지 않습니다.
- 홈 디렉토리 경로(`~/...`)나 특정 사용자 경로를 코드·설정에 하드코딩하지 않습니다.

## 공통 규칙

- **분류가 불확실하면** 작업 전에 사용자에게 어느 카테고리에 해당하는지 확인합니다.
- **최상위 디렉토리를 새로 만들 때**는 용도와 카테고리를 먼저 사용자와 확정합니다.
- **경로 참조**는 레포 루트 기준 상대 경로, POSIX 스타일(`/`)을 기본으로 합니다.
- **바이너리·대용량 파일**은 카테고리와 무관하게 커밋 전 필요성을 검토하고, 필요하면 Git LFS 도입을 논의합니다.
