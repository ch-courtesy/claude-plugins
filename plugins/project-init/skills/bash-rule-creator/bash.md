# Bash 지침

셸 명령을 실행할 때 따라야 할 규칙입니다. 개별 명령을 나열하는 대신, 명령의 **효과 범위와 가역성**을 기준으로 분류하고 카테고리별 규칙을 적용합니다.

## 명령 분류

명령을 실행하기 전에 어느 카테고리에 해당하는지 판단하고, 해당 카테고리의 규칙을 따릅니다. 애매하면 사용자에게 확인합니다.

### 1. 읽기 전용 (Read-only)

**특성**: 파일시스템·프로세스·네트워크 상태를 변경하지 않고 조회만 함.

**규칙**: 자유롭게 실행합니다. 다만 출력이 매우 길어질 수 있는 명령(`find /`, 전체 로그 덤프 등)은 범위를 좁힙니다.

**예**: `ls`, `pwd`, `grep`, `rg`, `git status`, `git log`, `ps`, `env`.

### 2. 로컬 변경·가역 (Local, reversible)

**특성**: 로컬 상태를 바꾸지만 되돌릴 수 있음.

**규칙**:
- 대상 경로를 확인한 뒤 실행합니다.
- 겹쳐쓰기(`>` 리다이렉트, `cp` 덮어쓰기, `mv` 대상 존재)가 발생하는지 미리 점검합니다.

**예**: `mkdir`, `touch`, `mv`, `cp`, 소규모 `sed -i`, 새 파일 생성.

### 3. 파괴적 (Destructive)

**특성**: 삭제·재귀적 변경·권한 변경처럼 되돌리기 어렵거나 불가능한 효과를 냄.

**규칙**:
- **사용자의 명시적 승인 없이 수행하지 않습니다.**
- 실행 전에 대상 경로를 절대경로로 명확히 하고, 영향 범위(재귀 삭제의 대상 파일 수 등)를 먼저 보여줍니다.
- 와일드카드 확장(`rm -rf *`)은 먼저 같은 패턴으로 `ls`를 실행해 대상을 검증합니다.

**예**: `rm`, `rm -rf`, `chmod -R`, `chown -R`, 디렉토리를 비우는 스크립트, DB `DROP`, 재생 불가능한 트렁케이션.

### 4. 권한 상승·시스템 (Privileged / system)

**특성**: 시스템 설정, 다른 사용자의 리소스, 패키지 관리 등 개인 권한을 벗어나는 영역에 영향을 줌.

**규칙**:
- `sudo`, 시스템 패키지 매니저(`apt`, `brew`, `dnf`), `systemctl`, 사용자·그룹 변경 명령은 사용자가 명시적으로 요청할 때만 실행합니다.
- 프로젝트 전용 환경(venv, 프로젝트 로컬 `node_modules` 등)으로 대체 가능한지 먼저 검토합니다.

**예**: `sudo ...`, `brew install`, `apt install`, `systemctl restart`.

### 5. 네트워크·외부 (Network / external)

**특성**: 외부 서비스에 영향을 주거나 외부 상태를 끌어와 결과가 결정적이지 않음.

**규칙**:
- 조회 성격(`curl -s`, `ping`, `dig`)은 비교적 자유. 인증된 엔드포인트는 키 노출에 주의.
- 변경 성격(POST/PUT/DELETE, `git push`, 배포 스크립트, 메시지 전송)은 사용자 승인 후 실행합니다.

## 작성·실행 원칙

- **경로 인용**: 공백·특수문자를 포함할 수 있는 경로는 항상 `"..."`로 감쌉니다.
- **절대경로 선호**: 여러 단계 명령·스크립트에서는 대상 경로를 절대경로로 명시합니다.
- **상호작용 명령 금지**: `git rebase -i`, `git add -i`처럼 에디터·프롬프트를 띄우는 명령은 비상호작용 환경에서 사용하지 않습니다.
- **오류 전파**: 스크립트에서는 `set -euo pipefail`을 기본값으로 고려합니다.
- **체이닝**: 의존 관계가 있는 명령은 `&&`로 연결하고, 독립 명령은 분리해 개별 실패를 구분합니다.
- **긴 명령은 스크립트로**: 한 줄 명령이 너무 길거나 재사용되면 파일로 분리합니다.

## 환경·셸 상태

- **`cd`의 부작용**: 세션의 cwd는 명령 간에 공유됩니다. 특정 디렉토리에서만 실행해야 하는 명령은 절대 경로나 `(cd path && cmd)` 서브셸로 감쌉니다.
- **환경변수 변경**: 전역 `export`는 의도가 분명할 때만. 임시 값은 `VAR=... command` 형태로 한 호출에만 적용합니다.
- **셸 이식성**: 기본은 POSIX 호환 문법. bash/zsh 전용 기능(배열, `[[ ]]`)을 쓰면 주석이나 shebang으로 명시합니다.

## 출력과 로깅

- **민감정보 출력 금지**: 환경변수 값, 자격증명, 토큰을 그대로 출력·로그에 남기지 않습니다. 필요하면 이름이나 마스킹된 형태로만 노출합니다.
- **stdout / stderr 분리**: 데이터는 stdout, 진행·경고 메시지는 stderr로 보냅니다.
- **긴 출력**: 수백 줄 이상이 예상되면 먼저 `head`, `tail`, 또는 필터로 범위를 좁힙니다.

## 대기·폴링

- **`sleep` 루프 지양**: 짧은 폴링은 가끔 필요하지만, 반복 실패에 `sleep` + 재시도로 대응하지 않고 원인을 찾아 수정합니다.
- **장시간 작업**: 수 분 이상 걸리는 명령은 백그라운드로 실행하거나 로그 파일에 리다이렉트하고, 진행 상태는 명시적으로 조회합니다.

## 공통 규칙

- **드라이런 먼저**: 파괴적·재귀적 명령은 가능하면 `--dry-run`, `-n`, 또는 먼저 `ls`/`grep`으로 대상을 확인한 뒤 실행합니다.
- **대상이 예상과 다르면** 실행하지 않고 사용자에게 보고합니다.
- **출처가 불분명한 스크립트**는 실행 전에 내용을 읽고, 신뢰할 수 없는 소스에서 오는 `curl ... | sh` 패턴은 피합니다.
